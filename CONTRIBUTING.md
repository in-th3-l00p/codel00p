# Contributing

codel00p is an open-source project in the blueprint stage. Contributions should
help clarify the architecture, prove the memory thesis, or reduce implementation
risk.

## Project priorities

Current priorities:

1. project memory design;
2. Hermes harness research;
3. local harness contracts;
4. CLI workflow design;
5. provider routing;
6. module protocol design.

Avoid starting with desktop or cloud features before the local memory and
harness loop is proven.

## Contribution style

Good contributions are:

- focused;
- easy to review;
- explicit about assumptions;
- aligned with local-first operation;
- clear about which subproject they affect.

Documentation should stay concise and technical. Prefer short files with clear
boundaries over one large document.

## Git history

Keep commits focused. A good commit should describe one coherent change.

Do not add co-author trailers unless the maintainer explicitly asks for them.

## Development status

The repository currently documents the technical direction. Implementation work
should begin with the memory model, harness research, and CLI prototype.
