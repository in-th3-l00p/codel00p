# Slice: v0.3.0 updater 403 fix

Date: 2026-06-15
Roadmap: Release & Self-Update follow-up.

## Goal

Ship a new release and fix `codel00p update` failing with `GitHub returned 403
Forbidden checking for updates` when the unauthenticated GitHub REST API rate
limit is exhausted.

## Diagnosis

`codel00p update --check` called
`https://api.github.com/repos/in-th3-l00p/codel00p/releases/latest`. The 403 was
GitHub REST API core rate limiting (`x-ratelimit-remaining: 0`), not a private
release or asset permission error. The public GitHub release redirect
`https://github.com/in-th3-l00p/codel00p/releases/latest` still resolves to the
latest release tag without consuming REST API quota.

## Design

- Resolve latest release by following the public `/releases/latest` redirect.
- Parse the final `/releases/tag/<tag>` URL for the tag name.
- Keep direct asset downloads unchanged.
- Bump the CLI crate to `0.3.0` and publish tag `v0.3.0`.

## Verification

- `codel00p update --check` from a local build no longer fails on REST API 403.
- `cargo test -p codel00p-cli update`
- Full `pnpm verify` before merge/tag.
