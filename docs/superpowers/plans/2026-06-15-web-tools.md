# Slice: Web fetch + search tools

Date: 2026-06-15
Roadmap: Milestone 1 (Agent parity) — web fetch/search tools.
Backlog/Initiative: Agent Parity Hardening (#3) + the built-in-tool-breadth gap
in the [Hermes gap analysis](../../initiatives/hermes-gap-analysis.md).

## Goal

Give the harness first-class web access so the agent can read documentation and
search the web mid-task — a capability Claude Code and Hermes both ship and
codel00p lacked. Two tools, behind a new `web` tool set and
`PermissionScope::Network`.

## Why this slice

Web access is a clear, self-contained agentic-capability gap: it adds a new tool
family without touching the turn loop, editing, or providers, and it is fully
offline-testable with `httpmock`.

## Design

`core/crates/codel00p-harness/src/web.rs`:

- **`web_fetch`** — GET a URL with `reqwest` (redirects, timeout, ~1 MB body
  cap), validate the scheme (http/https only — reject `file://`/`ftp://`),
  convert HTML to readable text (strip script/style/markup, collapse
  whitespace; pass plain text / JSON through), and return the text plus the
  final URL and status.
- **`web_search`** — query a pluggable search backend selected by env
  (`BRAVE_API_KEY`), returning `[{ title, url, snippet }]`. When unconfigured it
  returns a clear, actionable error rather than failing.

Both declare `PermissionScope::Network`. Registered in `tool_registry.rs` and a
new `web` tool set wired through the CLI (`agent/options.rs`, `agent/tooling.rs`)
so `--tool-sets web` (and `all`) include them. `httpmock` added as a harness
dev-dependency.

## Tests (test-first)

Offline `httpmock` cases: HTML→text rendering, JSON passthrough, redirect +
final-URL reporting, non-2xx error, body-size cap, URL-scheme validation, and
the `web_search` unconfigured-error path. No live network required.

## Out of scope

Browser/JS rendering, the `configuration.md` tool-set docs (owned by a separate
docs change), and richer search providers beyond the one pluggable backend.
